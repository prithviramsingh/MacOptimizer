Features

Everything your Mac needs
to stay fast.

MacOptimizer combines five powerful tools in a clean, native interface. No Electron, no bloat — just pure Swift.

📊

Real-Time Process Monitor

Polls ps every 5 seconds and tracks CPU & RAM usage per process. Automatically flags resource hogs exceeding 20% CPU or 500 MB RAM with color-coded severity.

🧹

System Cleanup

Scans and cleans ~/Library/Caches, system Logs, Xcode DerivedData & Simulators, iOS device backups, and flushes the DNS cache. Safe dry-run mode on by default.

🚀

Startup Item Manager

Lists all LaunchAgents from ~/Library/LaunchAgents and /Library/LaunchAgents. Toggle any item on/off without leaving the app using launchctl load/unload.

💡

Smart Suggestions Engine

Cross-references live process data against a built-in knowledge base of 13+ common macOS processes. Delivers severity-ranked, actionable fix cards in plain language.

🧠

Process Knowledge Base

Knows exactly what kernel_task, mds_stores, WindowServer, Chrome, Slack, Xcode, Node, and more mean — and tells you what to do about them.

📈

History & Trend Detection

Tracks process CPU samples over time and surfaces sustained hogs — processes that average high CPU across multiple polling intervals — even if they spike intermittently.

How It Works

Simple by design.

MacOptimizer runs entirely on your Mac — no cloud, no accounts, no data collection. Everything happens locally.

1

Launch the app

Open MacOptimizer from Xcode or the command line. It immediately starts monitoring your system with a 5-second polling loop — no setup required.

2

Spot the hogs on the Dashboard

The Dashboard shows live stat cards for Active Processes, Resource Hogs, and Suggestions. The Top Resource Consumers table highlights your most expensive processes at a glance.

3

Dive deeper with dedicated views

Switch to Processes for the full sortable list. Use Cleanup to scan for recoverable disk space (always in dry-run first). Check Startup Items to prune what launches at login.

4

Act on Suggestions

The Suggestions tab analyses your live processes against the knowledge base and surfaces ranked cards — Critical, Warning, or Info — each with a specific, plain-language fix you can apply immediately.

Knowledge Base

Knows the processes that matter.

MacOptimizer ships with expert-curated knowledge about the most common CPU and RAM culprits on macOS.

kernel_task

High CPU means macOS is throttling the processor to protect against heat. MacOptimizer tells you to cool your Mac rather than panic.

Installation

Up and running in minutes.

MacOptimizer is a Swift Package — build and run it directly from Xcode or the terminal. No third-party dependencies.

Requirements

macOS 13 Ventura or later
Xcode 15+ (for IDE build)
Swift 5.9+ toolchain
Apple Silicon or Intel

  1. Clone the repository
git clone https://github.com/prithviramsingh/MacOptimizer.git
cd MacOptimizer
  1. Build the app
swift build -c release
  1. Run it
# Using the helper script
./run.sh

# Or directly
.build/release/MacOptimizer

💡 Tip: The first build may take a moment as Swift compiles all sources. Subsequent builds are incremental and much faster.

  1. Clone the repository (or download the ZIP from GitHub)
  2. Open Xcode 15 or later
  3. Choose File → Open and select Package.swift in the project folder
  4. Xcode automatically resolves the Swift Package — wait for indexing to complete
  5. Select the MacOptimizer scheme in the toolbar and press ⌘R to build & run

💡 Tip: If you see a permission prompt the first time the app reads /Library/LaunchAgents or runs launchctl, click Allow.

The included distribute.sh script builds a release binary and bundles it into a .app package you can share or install to /Applications.

# Make executable (first time only)
chmod +x distribute.sh

# Build and package
./distribute.sh

The script creates MacOptimizer.app in the project root. Drag it into /Applications to install system-wide.

Architecture

Clean, layered Swift code.

MacOptimizer follows a services → models → views layering pattern, making it easy to extend or contribute.

MacOptimizer/
├── Package.swift — Swift Package Manifest (macOS 13+, no deps)
├── MacOptimizer.entitlements — App sandbox entitlements
├── run.sh — Convenience build & run script
├── distribute.sh — Packages a .app bundle for distribution
└── Sources/MacOptimizer/
├── MacOptimizerApp.swift — @main, injects all EnvironmentObjects
├── ContentView.swift — NavigationSplitView with 5 nav items
├── Models/
│ ├── ProcessInfo.swift — ProcessSnapshot + ProcessHistory structs
│ ├── CleanupItem.swift — Represents a scannable cleanup target
│ └── Suggestion.swift — Severity-ranked suggestion model
├── Services/
│ ├── ShellRunner.swift — Async shell command execution
│ ├── ProcessMonitor.swift — Polls ps every 5s, tracks history
│ ├── SystemCleaner.swift — Scan & clean caches / logs / DerivedData
│ ├── StartupItemsService.swift— LaunchAgents via launchctl
│ └── KnowledgeBase.swift — 13+ process entries, generates Suggestions
└── Views/
├── DashboardView.swift — Stat cards + top consumers + suggestions preview
├── ProcessesView.swift — Sortable full process list
├── CleanupView.swift — Scan results + dry-run / clean actions
├── StartupItemsView.swift — Toggle launch agents
└── SuggestionsView.swift — Severity-ranked suggestion cards

FAQ

Common questions.

Is MacOptimizer safe to run?

Yes. By default, the Cleanup module runs in dry-run mode and will only report what it would delete — it won't touch any files until you explicitly confirm. The Startup Items module uses the standard launchctl API, which is fully reversible.

Does it send any data off my machine?

No. MacOptimizer has zero network calls and zero analytics. All monitoring, analysis, and cleanup happen entirely on your local machine. The source code is open — you can verify this yourself.

Why does Cleanup need permission to access certain folders?

macOS sandboxes apps by default. When MacOptimizer tries to read /Library/LaunchAgents or certain system cache paths, macOS may prompt you for permission. These prompts are shown by the OS and granting them does not give the app any elevated privileges beyond reading those specific paths.

Can I kill processes from within the app?

The Suggestions view shows a Kill Process button for processes that are marked safeToKill: true in the knowledge base (e.g. Chrome Helper, Slack, coreaudiod). System-critical processes like kernel_task and WindowServer are never killable from the app.

The app says a process is a "resource hog" — what does that mean?

A process is flagged as a resource hog if it exceeds 20% CPU or 500 MB RAM in a polling sample. These thresholds reflect typical signs of runaway behaviour on a modern Mac, but your mileage may vary — a heavy Xcode build, for example, will intentionally spike above these numbers.

How do I add a new process to the knowledge base?

Open Sources/MacOptimizer/Services/KnowledgeBase.swift and add a new entry to the db dictionary using the ProcessKnowledge struct. Each entry takes a display name, description, optional CPU/RAM suggestions, and a safeToKill flag. Pull requests are welcome!

Does it work on Apple Silicon and Intel Macs?

Yes. MacOptimizer is compiled by Xcode's standard Swift toolchain, which produces a universal binary that runs natively on both Apple Silicon (M1/M2/M3/M4) and Intel processors.
